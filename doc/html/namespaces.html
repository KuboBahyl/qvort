Modules List

Here is a list of all documented modules with brief descriptions:
cdataHold main variable globally, all other modules will need this module included to access the filament/particle arrays and other important variables also contains important routines for initialising random number generator and reading in runfile
diagnosticDiagnostic routines purely for vortex filament, if acting as a magnetic field diagnostics done within the module mag, particle diagnostics contained within quasip.mod
forcingAll the routines to employ forcing in the code note this is different to the normal velocity for a vortex system normal fluid enters as whereas forcing is directly added to vortex velocity i.e.
generalGeneral routines e.g. derivatives, vector products, distance evaluation this is used by all other modules (excluding cdata)

initialModule which contains all the routines/call to routines which setup and restart the code. Initial conditions for filament set in run.in for more information on the options see Initial condition for filament
KSmodelAll the routines required to setup and subsequently calculate the KS model at a given position and time. This is used as as a normal fluid in the code. The KS model is a model of a turbulent like velocity field created throught the summation of random fourier modes

where , is the number of modes, and are their wave vectors and frequencies

lineAll routines which alter the geometry of the vortex filament/flux tube should be contained in this module
magAll routines related to the filaments acting as magnetic flux tubes are in this module
normal_fluidNormal fluid component in the equation of motion, if the filament acts as a vortex velocity enters as if the filament is a magnetic flux tube or a material line then the normaly velocity is used directly. for further information on the normal fluid options see Normal fluid velocity field
outputThe main output routines from the code/specifc modules may contain there own output, however main routines for printing structures should be in here
particlesThe particles aspect of the code. At present this evolves either fluid or inertial particles in the code. The type of particle is set by the particle_type parameter in run.in Fluid/Inertial particles use the normal fluid velocity. The initial setup of the particles is set throught the initp parameter
periodicAll the routines used to keep the code periodic
quasipThe quasi particles aspect of the code. The superfluid velocity if used to evolve the particle using the stiff_solver and hamiltonian module. The initial setup of the particles is set throught the initg parameter
smoothingAll routines used to smooth the field onto the mesh, using tree methods if the code is running as a vortex filament this is simply the vorticity if a flux tube this smooths the magnetic field
sphSPH smooth paricle hydrodynamics
stiff_solverAll routines to solve stiff ode problems in particular this is used for reflection of quasi particles
timestepTimestepping and velocity routines are contained in this module
treeAll the tree routines used by the code are contained within this module, pointers are used to create a linked list, be very careful to avoid memory leaks!
