Functions/Subroutines

subroutine construct_tree ()
 An array to set the main node of the tree and then call the recurisive subroutine build_tree this is called in the main program.
recursive subroutine build_tree (vtree)
 called by above, build the tree, continue to subdivide until a cell is empty or contains only one particle - octree structure used each new cell is created by the routine below new_tree
subroutine new_tree (vtree, posx, posy, posz, width, loc_pcount, parray)
 allocate a new cell, populate with particles and calculate centre of mass, total circulation vector and magnetic field vector
recursive subroutine empty_tree (vtree)
 empty out the tree structure to avoid a memory leak - repeatedly calls dead_tree
subroutine dead_tree (vtree)
 deallocate a tree
subroutine pclose_tree
 find the closest particle using the tree structure, theoretically should be NlogN repeatedly calls closest_tree
recursive subroutine closest_tree (i, vtree)
 recurisve routine used by pclose_tree to find the closest particle to i
recursive subroutine tree_walk (i, vtree, shift, u)
 get the tree code approximation to the biot savart integral, giving the induced velocity at particle i due to all other vortices, accepts an arguement shift, which allows you to shift the position of the vortices for periodic boundary conditions
recursive subroutine tree_walk_general (x, vtree, shift, u)
 a more general form of above, where a position is an input this is used to calculate the superfluid velocity at mesh points
